# Codex 0.120 Compatibility Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Re-anchor `codex_feishu` on `codex-cli 0.120.0`, verify that the current thin-bridge contract still holds, and selectively absorb only the upstream changes that materially improve bridge stability.

**Architecture:** Keep the bridge thin and native-first. This upgrade does not expand the `/codex` product surface to match every new upstream Codex feature; it updates the verified compatibility baseline, hardens runtime/docs/tests around `0.120.0`, and runs one narrow evaluation of `codex exec` prompt-plus-stdin support to decide whether it reduces bridge prompt assembly risk without changing user-facing semantics.

**Tech Stack:** Node.js ESM, `node:test`, existing bridge runtime compatibility probes, shell install scripts, Markdown governance docs
